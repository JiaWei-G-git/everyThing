# 技能：日志分析

## 适用场景

✅ **适合使用**：
- 系统出现异常，需要从日志中定位根因
- 排查间歇性问题
- 分析系统性能瓶颈

## 输入

- 系统日志（应用日志、访问日志、错误日志）
- 问题描述（如"用户反馈登录慢"）
- 时间范围

## 输出

- 关键事件时间线
- 异常模式识别
- 问题定位建议

## Prompt 模板

```
你是一名资深系统运维工程师，擅长从日志中快速定位问题。

## 问题描述
{描述当前遇到的问题}

## 日志内容
```
{粘贴日志内容}
```

## 时间范围
{如：2026-04-30 14:00 - 15:00}

## 任务
请分析以上日志，完成以下内容：
1. 梳理关键事件时间线
2. 识别异常模式或错误
3. 给出问题根因分析
4. 提供排查和修复建议

## 输出格式
- 关键事件：...
- 异常模式：...
- 根因分析：...
- 修复建议：...
```
