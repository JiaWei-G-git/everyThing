# Dashboard v0.2.0 UI 设计优化

> 基于 ui-ux-pro-max 设计原则，为 VS Code Webview Dashboard 提供 UI 优化指导。

---

## 1. 图标系统（Critical）

### 问题
当前大量使用 emoji（🤖 🔍 ⭐ ⚙ 📋 🕐 🔥 🚀 等）作为结构性图标。

### 规则
ui-ux-pro-max: `no-emoji-icons` — emoji 是字体依赖的，跨平台不一致，无法通过设计 token 控制。

### 方案
用内联 SVG 替代所有 emoji，统一风格：
- Stroke width: 1.5px
- Corner: rounded (2px)
- 尺寸 token: sm(16px) / md(20px) / lg(24px)

| 原 Emoji | 替换为 SVG 图标 | 语义 |
|----------|----------------|------|
| 🤖 | robot / sparkles | AI/Logo |
| 🔍 | search | 搜索 |
| ⭐ | star | 收藏 |
| ⚙ | settings | 设置 |
| 📋 | clipboard-list | 角色/集群 |
| 🕐 | clock | 最近使用 |
| 🔥 | flame | 热门/常用 |
| 🚀 | rocket | 新手上路 |
| 📝 | file-text | 写文档 |
| 🔧 | wrench | 查故障 |

## 4. 对比度检查

当前已使用 VS Code CSS 变量（语义 token），这是正确的做法。但需确保：

| Token | 用途 | 建议 |
|-------|------|------|
| `--vscode-foreground` | 主文字 | 已在用，对比度由 VS Code 保证 |
| `--vscode-descriptionForeground` | 次要文字 | 已在用 |
| `--vscode-textLink-foreground` | 强调色/链接 | 已在用 |
| 自定义 `--card-bg` | 卡片背景 | 确保与 `--vscode-editor-background` 有足够对比度 |

当前 `--card-bg` 使用的是 `--vscode-editor-inactiveSelectionBackground`，这在某些主题下可能太亮。建议改为更保守的选择：

```css
:root {
  --card-bg: var(--vscode-editor-inactiveSelectionBackground, #3a3d41);
  /* 备选：如果对比度不够，可以添加透明度层 */
}
```
