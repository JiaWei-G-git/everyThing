# AI 技能知识库 v0.2.0 — 产品与 UI 规格说明

> 本文档面向 UI/UX 重设计输入。它描述**做什么、长什么样、怎么用**,不绑定现有实现细节。设计师可以从零产出新的视觉 / 交互方案。
>
> 范围:VS Code 扩展 webview 内呈现的 Dashboard 单面板。
>
> 容器:VS Code 侧边栏(右侧或左侧),最小宽度 ~550 px。

---

## 1. 一句话定位

把团队成员各自摸索出的 AI 协作经验(Prompt、工作流、检查清单),沉淀成**可复用、可分发、可一键安装**的 "技能" 资产,让任何一位团队成员都能在 1-2 分钟内找到能解决当前问题的 AI 能力。

---

## 2. 目标用户与典型动机

| 用户 | 来这里干什么 |
|---|---|
| 产品经理 / 业务分析师 | 找会议纪要提取、需求拆解类技能 |
| 前端 / 后端开发 | 找 API 设计、组件生成、bug 定位类技能 |
| 测试工程师 | 找测试用例生成、质量审查类技能 |
| 运维工程师 | 找故障排查检查清单、CI/CD 配置类技能 |
| 架构师 | 找架构设计、代码审查类技能 |
| 新员工 | 顺着"新手上路"做一遍,理解团队 AI 协作约定 |

**关键:用户来时通常已经有一个具体任务**(写周报 / 排查告警 / 设计 API),不是在闲逛。UI 必须帮他在 30 秒内"对号入座"。

---

## 3. 核心概念词典

| 中文 | 字段 | 含义 | UI 中的形态 |
|---|---|---|---|
| 集群 | Cluster | 一个工程领域(如"03-开发工程"),包含多个智能体 | 卡片(角色视图) |
| 智能体 | Agent | 一个角色化的 AI 助手定义,挂多个技能 | 列表(集群详情页) |
| 技能 | Skill | 可单独使用的 AI 能力单元(Prompt + 步骤) | 列表项 / 详情页 |
| 检查清单 | Checklist | 特殊类型的技能,内容是"步骤 + 复选框" | 详情页特殊渲染 |
| 场景 | Scenario | 跨集群的横切标签(如"写文档"、"查故障") | 卡片(场景视图) |
| 行业知识包 | Industry Pack | 用户可加载的外部技能包(如电力行业) | 设置页条目 |
| 使用统计 | Usage Stats | 本地记录用户用过哪些 skill / 哪些 scenario | "最近使用"、"我的常用场景" |
| 收藏 | Favorite | 用户标记的常用 skill | 顶部 Tab "收藏" |

**类型枚举**:
- `Skill.type`: `'skill' | 'checklist'`
- `Skill.source`: `'core' | 'industry' | 'imported'`
- `ViewMode`: `'role' | 'scenario'`(首页双视图 Tab)

## 4. 信息架构

```
顶部导航(4 个 Tab,常驻)
├─ 🏠 首页 /dashboard       ← 主入口,90% 使用从这里发起
├─ 🔍 搜索 /search          ← 关键词找技能
├─ ⭐ 收藏 /favorites       ← 个人技能库
└─ ⚙ 设置 /settings        ← 路径配置 + 行业包 + 导入

二级页面(从一级跳转,无独立 Tab):
├─ /cluster   集群详情(从首页角色卡跳)
├─ /skill     技能详情(从任何技能列表项跳)
└─ /scenario  场景详情(从场景卡跳,v0.2.0 仅占位)
```

## 5. 页面规格

### 5.1 首页 `/dashboard`

最复杂的页面,也是用户停留时间最长的。当前结构(按视觉从上到下):

```
┌─────────────────────────────────────────┐
│ Logo + 标题 + 版本号                     │  -- 5.1.1
├─────────────────────────────────────────┤
│ 欢迎语                                   │
├─────────────────────────────────────────┤
│ 🔍 搜索框(点击跳搜索页)                  │  -- 5.1.2
├─────────────────────────────────────────┤
│ ┌──┐ ┌──┐ ┌──┐ ┌──┐ 4 个统计卡        │  -- 5.1.3
│ │6 │ │7 │ │8 │ │8 │ 集群/智能体/技能/场景│
│ └──┘ └──┘ └──┘ └──┘                    │
├─────────────────────────────────────────┤
│ [按角色浏览] [按场景浏览]   ← 双视图 Tab │  -- 5.1.4
├─────────────────────────────────────────┤
│ ╔ Tab 内容 ════════════════════════════╗│
│ ║ Role 视图: 6 个集群卡片              ║│  -- 5.1.5
│ ║ Scenario 视图: 8 个场景卡片          ║│  -- 5.1.6
│ ╚══════════════════════════════════════╝│
├─────────────────────────────────────────┤
│ 🕐 最近使用(列表,最多 4 条)             │  -- 5.1.7
├─────────────────────────────────────────┤
│ ⚡ 快速创建(3 个虚线卡)                 │  -- 5.1.8
├─────────────────────────────────────────┤
│ 🔥 我的常用场景(列表,基于使用统计)      │  -- 5.1.9
├─────────────────────────────────────────┤
│ 🚀 新手上路(4 步骤,引导卡)              │  -- 5.1.10
└─────────────────────────────────────────┘
```

#### 5.1.1 Logo 区
- 字段:`logo-icon`(图标)、`logo-title`(产品名)、`logo-version`(版本号 v0.2.0)
- 居中,图标 32px,标题 16px,版本号 11px

#### 5.1.2 搜索框
- 单行输入,占位符"快速搜索技能或智能体..."
- **整行点击都跳到 `/search` 页**(不能在首页直接搜)
- 当前左侧有🔍图标,这是 emoji,设计稿建议改为 SVG / 图标字体

#### 5.1.3 4 个统计卡
- 字段:cluster / agent / skill / scenario 数量
- 排版:横向四列(屏小可换 2x2 网格)
- 每卡:大数字 + 标签 + 可选副文案("+2 较上月"是占位的虚假数据,真实环境无来源,**重设计可考虑去掉副文案**)

#### 5.1.4 双视图 Tab
- 两个 button 平分宽度,选中态 + active 背景
- 切换时 5.1.5 / 5.1.6 切换,**其他区域不变**
- 当前激活态用 VS Code 主题的 `bg-active`(深蓝/紫)

#### 5.1.5 Role 视图(默认)
- 6 个集群卡片,网格(屏宽决定列数,推荐 2-5 列响应式)
- 每卡显示:
  - 集群图标(emoji,可换 SVG)
  - 集群名(如"03-开发工程"),**开发工程置顶**且带"核心"红徽章
  - 描述(一行)
  - 底部 meta:`X 个智能体 · Y 个技能` + "进入 →"
- 整卡可点击,跳 `/cluster?id=xxx`
- 关键视觉差异:**开发工程**有特殊突出(底色渐变 / 描边强调 / "核心"徽章)

#### 5.1.6 Scenario 视图
- 8 个场景卡,网格(推荐 4 列 x 2 行,响应式)
- 8 个场景固定:写文档 / 查故障 / 写代码 / 搞数据 / 做设计 / 管项目 / 测试质量 / 知识检索
- 每卡居中显示:大图标(28px) + 名称 + 一行说明 + 技能数(如"3 个技能",右下角小字蓝色)
- 整卡可点击,跳 `/scenario?id=xxx`(v0.2.0 仅 Toast 占位,后续做)

#### 5.1.7 最近使用
- 区头:左"🕐 最近使用",右"全部历史"按钮
- 列表项:文档图标 + 技能名 + meta(`X 分钟前 · 智能体名 · 集群名`)
- 整行可点击进 `/skill`
- 空状态:"暂无使用记录"灰字

#### 5.1.8 快速创建
- 区头:"⚡ 快速创建"
- 3 个虚线卡(网格):写周报 / 会议纪要 / 项目材料
- 点击当前仅 Toast 占位,**重设计:这 3 个卡可以改为产品中要重点引导的高频任务,虚线代表"模板/快捷入口"语义**

#### 5.1.9 我的常用场景
- 区头:"🔥 我的常用场景"
- **空时整段不显示**(当前实现)
- 有数据时:列表 排名 + 场景名 + "X 次"
- 数据来自本地 `usageStats.scenarioUsage`,在用户每次打开 skill 时累加

#### 5.1.10 新手上路
- 卡片化,4 步骤:
  1. 了解智能体和集群 / 浏览六大工程集群
  2. 选择技能并安装 / 找到适合的技能
  3. 执行工作流 / 按步骤完成任务
  4. 收藏常用技能 / 建立个人技能库
- 当前步骤 1 的副文案不准确(plan 里写的是"按角色或按场景找到技能",更贴合 v0.2.0)
- 视觉:序号 + 标题 + 副标题,可考虑加进度引导(已使用了哪个步骤)

### 5.3 技能详情 `/skill` (最关键的页面之一)

两种渲染模式,根据 `skill.type`:

#### 模式 A:普通技能(`type: skill`)

```
┌─────────────────────────────────────┐
│ ← 返回                              │
│ <技能名>                            │
│ 版本 1.0.0 · 日期 · (无 type 徽章)  │
│ #tag #tag  [写文档] [管项目]        │  -- 普通 tag + 场景 tag
├─────────────────────────────────────┤
│ ⏱ 预计 10-15 分钟 · 🔗 可独立使用    │
├─────────────────────────────────────┤
│ [📋 复制 Prompt] [⬇ 安装到 Claude]   │
│ [⬇ 安装到 Cursor] [☆/⭐ 收藏]        │
├─────────────────────────────────────┤
│ Markdown 渲染的技能内容             │
│ (## 标题 / ``` 代码块 / 列表 等)    │
└─────────────────────────────────────┘
```

#### 模式 B:检查清单(`type: checklist`)

```
┌─────────────────────────────────────┐
│ ← 返回                              │
│ <技能名> [📋 检查清单 徽章]          │
│ tag + 场景 tag                      │
│ [复制] [安装] [收藏] (同上)          │
├─────────────────────────────────────┤
│ ━━━━━━━━━━━━━━━━━ 25% (3/12)        │  -- 总进度条
├─────────────────────────────────────┤
│ ▼ 步骤 1:确认故障现象      1/3      │
│   ☑ 终端完全离线                    │
│   ☐ 仅召测失败                      │
│   ☐ 采集率下降                      │
├─────────────────────────────────────┤
│ ▶ 步骤 2:检查网络层        0/4      │
├─────────────────────────────────────┤
│ ▶ 步骤 3:检查业务层        0/4      │
├─────────────────────────────────────┤
│ (可选)✅ 全部完成                   │
└─────────────────────────────────────┘
```

- 步骤可折叠,默认展开"第一个或还有未勾选的步骤"
- checkbox 状态 v0.2.0 **不持久化**(刷新重置),设计上这是"陪跑工具"
- 进度条用主题 success 色,平滑过渡
- 全部勾选后底部出现"✅ 全部完成"绿色横幅

## 6. 数据模型

### Skill
```typescript
interface Skill {
  id: string;
  name: string;
  description: string;
  type: 'skill' | 'checklist';
  version: string;
  date: string;
  parentAgent: string;
  cluster: string;        // 集群 id,如 "03-开发工程"
  standalone: boolean;
  tags: string[];          // 普通标签
  scenarioTags: string[];  // 场景标签(写文档 / 查故障...)
  source: 'core' | 'industry' | 'imported';
  industryPackage?: string;
  input?: string;
  output?: string;
  estimatedTime?: string;  // "10-15分钟"
  filePath: string;
  content: string;         // Markdown 正文
}
```

### Cluster / Agent
```typescript
interface Cluster {
  id: string;       // 如 "03-开发工程"(永远以 "数字-" 前缀)
  name: string;     // 不带前缀
  description: string;
  agents: string[]; // agent id 列表
  skills: string[]; // skill id 列表
}

interface Agent {
  id: string;
  name: string;
  description: string;
  cluster: string;
  roles: string[];
  skills: string[];
  filePath: string;
}
```

### 用户状态
```typescript
interface UsageStats {
  skillUsage: Record<string, number>;       // skillId → 次数
  scenarioUsage: Record<string, number>;    // scenarioTag → 次数
  lastUsed: number;
}

interface DashboardStats {
  clusterCount: number;
  agentCount: number;
  skillCount: number;
  scenarioCount: number;
}

type ViewMode = 'role' | 'scenario';
```

## 7. 关键交互流程

### 流程 A:用户带任务进来(高频)

```
打开侧边栏
  → 首页统计卡瞄一眼总量
  → 选择 "按场景浏览" Tab
  → 点 "查故障" 卡片(显示 3 个技能)
  → 进 /skill (普通) 或 /skill (checklist)
  → 复制 Prompt 或安装到 Claude
  → 收藏 (☆ → ⭐)
```

### 流程 B:浏览式探索

```
打开侧边栏
  → 默认 Role 视图,看 6 个集群
  → 点击 "03-开发工程" 集群卡(置顶)
  → 看到智能体 + 各自的技能
  → 点击某个技能进详情
```

### 流程 C:执行检查清单(陪跑)

```
进 /skill (type=checklist)
  → 看到顶部进度条 0%
  → 第 1 步默认展开
  → 边做实际工作边勾选 checkbox
  → 进度条实时更新
  → 全部勾完出现"全部完成"
  → ⚠ 关闭面板会丢失勾选状态(v0.2.0 无持久化)
```

### 流程 D:首次使用(新员工)

```
打开 → 看"新手上路" 4 步骤
  → 步骤 1 引导到选择浏览方式(角色/场景)
  → 步骤 2 引导到安装一个技能
  → ...
  → 完成上手
```

## 8. 视觉风格

### 8.1 配色 — 强制使用 VS Code 主题变量

| 用途 | 变量 | Fallback(暗色) |
|---|---|---|
| 主背景 | `--vscode-editor-background` | `#1e1e1e` |
| 二级背景 | `--vscode-sideBar-background` | `#252526` |
| Hover 背景 | `--vscode-list-hoverBackground` | `#2a2d2e` |
| 选中背景 | `--vscode-list-activeSelectionBackground` | `#094771` |
| 主文本 | `--vscode-foreground` | `#cccccc` |
| 副文本 | `--vscode-descriptionForeground` | `#858585` |
| 强调色 | `--vscode-textLink-foreground` | `#3794ff` |
| 边框 | `--vscode-panel-border` | `#3c3c3c` |
| 卡片背景 | `--vscode-editor-inactiveSelectionBackground` | `#3a3d41` |
| 成功 | `--vscode-testing-iconPassed` | `#4ec9b0` |
| 警告 | `--vscode-editorWarning-foreground` | `#cca700` |
| 错误 | `--vscode-editorError-foreground` | `#f14c4c` |

**重要约束:不能用任意品牌色,必须用 VS Code 变量,使插件在用户切换主题时跟随**(暗黑/明亮/各种社区主题)。

### 8.2 间距 / 字号

- 字号:11/12/13/16 px 四档
- 行高:1.5
- 圆角:6 px (var --radius)
- 间距:8 / 12 / 16 px (var --gap = 12)
- 卡片 padding:14-16 px
- 最小宽度:550 px (用户调小不会破布局)

### 8.3 当前用了大量 emoji 作图标

```
🤖 🔍 ⭐ ⚙ 📋 🕐 🔥 🚀 ⚡ 📝 🔧 💻 🗄️ 📐 📊 ✅ 📚 🎯 🏷️ 📦 📥 📂 ⏱ 🔗
```

设计建议:**全部替换为线条 SVG 图标**(stroke 1.5px,圆角端点),保持跨平台/主题一致性。已有一份候选映射在 `docs/design-v0.2.0-ui.md`。

## 10. 设计原则提示(给重设计的硬约束)

1. **VS Code 集成感**:必须用 VS Code 变量,不要写死颜色;不要用大圆角(那像 SaaS web)
2. **侧边栏窄**:实际宽度可能 300-600 px,不是宽屏 dashboard,所有横向布局要可折叠到单列
3. **文本可读**:VS Code 用户多数眯眼看代码,文本不能小于 11 px
4. **零等待**:点击 Tab 切换、点击卡片跳页,不要加无意义动画延迟(>300ms)
5. **键盘可达**:VS Code 用户重度键盘党,所有交互应可 Tab + Enter
6. **暗色优先**:VS Code 用户 ~80% 用暗色主题,先在暗色下设计
