# AI 技能知识库平台 —— 项目概述

## 一、项目背景

随着 AI 编程助手（Claude Code、Cursor、GitHub Copilot 等）在研发团队中的普及，团队成员各自积累了一套与 AI 协作的 Prompt、工作流和最佳实践。但这些知识分散在个人笔记、聊天记录和口头分享中，存在以下问题：

1. **无法复用**：A 同事写了一个很好的"API 设计 Prompt"，B 同事不知道，重复造轮子
2. **质量参差**：每个人写的 Prompt 质量不一，新人很难写出有效的指令
3. **没有积累**：人员流动导致团队 AI 协作经验流失
4. **工具割裂**：同样的能力，在 Claude Code 和 Cursor 中需要写两套 Prompt

## 二、项目目标

搭建一套**公司级 AI 技能知识库平台**，实现：

1. **集中管理**：将团队与 AI 协作的最佳实践沉淀为可复用的"技能"和"智能体"定义
2. **按角色分发**：产品经理、前端开发、测试工程师等不同角色，能快速找到适合自己工作的 AI 能力
3. **一次定义，多平台使用**：以 Markdown 为核心格式，支持安装到 Claude Code、Cursor、通用 AI 工具
4. **按需取用**：可以安装完整的"智能体"（一组技能），也可以只取用一个"技能"
5. **持续演进**：技能库本身是一个活的系统，随团队实践持续更新

## 三、项目范围

### 3.1 包含内容

| 维度 | 说明 |
|:---|:---|
| 智能体集群 | 6 大集群，覆盖 Web 应用项目全生命周期 |
| 智能体 | 每个集群下 2-4 个智能体，共约 17 个 |
| 技能 | 每个智能体下 2-6 个技能，共约 50-80 个 |
| 索引系统 | 总索引（按角色/场景）+ 集群索引 + 智能体技能清单 |
| 平台适配 | Claude Code、Cursor、通用 Prompt 三种格式 |

### 3.2 不包含内容

- 不开发 Web 界面或管理系统（第一阶段）
- 不做自动格式转换工具（第一阶段用手工维护）
- 不包含非 Web 应用领域的技能（如嵌入式、数据科学）
- 不包含特定业务领域的知识（如金融、医疗业务规则）

## 四、术语定义

| 术语 | 英文 | 定义 |
|:---|:---|:---|
| 智能体集群 | Agent Cluster | 同一领域的一组智能体的集合，如"需求工程集群" |
| 智能体 | Agent | 一个角色化的 AI 能力包，包含角色定义和若干技能 |
| 技能 | Skill | 一个可独立执行的 AI 能力单元，包含 Prompt 模板和执行步骤 |
| 索引 | Index | 帮助用户快速定位智能体和技能的导航文件 |
| 平台适配 | Platform Adapter | 将通用技能内容转换为特定 AI 工具可用格式的规则 |

## 五、目标用户

| 用户角色 | 使用场景 | 主要需求 |
|:---|:---|:---|
| 产品经理 / 业务分析师 | 需求采集、需求分析、原型设计 | 找到需求相关的智能体和技能，提升需求产出质量 |
| 前端开发工程师 | 组件开发、页面实现、交互优化 | 找到前端开发相关的技能模板，加速编码 |
| 后端开发工程师 | API 设计、业务逻辑、数据库设计 | 找到 API 设计和后端开发技能，规范接口 |
| 测试工程师 | 测试设计、测试执行、缺陷管理 | 找到测试相关技能，提升测试覆盖率和效率 |
| 运维工程师 | 部署上线、监控告警、故障处理 | 找到运维相关技能，减少人为失误 |
| 技术负责人 / 架构师 | 架构设计、技术选型、代码审查 | 找到架构设计和审查技能，统一团队标准 |

### 特殊用户场景

**新入职员工**：不属于角色导航的 6 个核心角色。新入职员工通过"新手指引"入口了解团队 AI 工作流，而非通过角色索引定位技能。

## 六、成功指标

| 指标 | 目标值 | 测量方式 |
|:---|:---|:---|
| 技能数量 | MVP 阶段 15-20 个，完整覆盖阶段 40-50 个 | 统计 skills/ 目录下文件数 |
| 周活跃用户 | 团队 50% 以上成员每周至少使用 1 次 | 定期调研+Git提交统计（估算引用/下载次数） |
| 查找效率 | 从"我想做 X"到"找到对应技能"平均 < 2 分钟 | 用户测试 |
| 技能复用率 | 同一个技能被不同成员使用 ≥ 3 人 | 调研统计 |
| 满意度 | 使用者满意度 ≥ 7/10（项目最终目标，分阶段递进：阶段二≥6/10，阶段三≥7/10，阶段四≥8/10） | 季度问卷 |
