# AI 技能知识库平台 —— 智能体定义规范

## 一、文件定位

智能体定义文件描述了一个**角色化的 AI 能力包**：
- 这个角色是谁（定位）
- 这个角色能做什么（技能清单）
- 如何把这个角色安装到 AI 工具中（安装说明）

智能体文件本身**不包含技能的具体执行细节**，只提供索引和安装指引。具体执行细节在技能文件中定义。

## 二、文件命名

```
{中文智能体名}智能体.md

示例：
- 需求分析智能体.md
- 原型设计智能体.md
- 后端开发智能体.md
```

## 三、文件位置

```
clusters/{集群文件夹}/

示例：
clusters/01-需求工程/需求分析智能体.md
clusters/02-设计工程/原型设计智能体.md
```

## 四、Frontmatter 规范

```yaml
---
name: "需求分析智能体"                    # 智能体显示名称
description: "需求深度解析与文档化专家"    # 一句话描述
type: "agent"                           # 固定值：agent
version: "1.0.0"                        # 语义化版本
date: "2026-04-29"                    # 最后更新日期
cluster: "需求工程"                      # 所属集群
roles:                                  # 适用角色（用户身份）
  - "产品经理"
  - "业务分析师"
  - "系统分析师"
skills-count: 5                         # 包含的技能数量（可选，用于统计）
---
```

## 五、内容模块规范

智能体文件必须包含以下模块，按此顺序排列：

### 模块 1：角色定位

**目的**：让用户在 10 秒内判断这个智能体是否适合自己当前的需求。

**内容要求**：
- **一句话定义**：用 1 句话说明这个智能体是什么角色（控制在 30 字以内）
- **详细说明**：2-3 句话，说明这个角色的职责范围、输入输出、工作边界

**示例**：

```markdown
## 角色定位

**一句话定义**：需求深度解析与文档化专家，将原始需求转化为结构化的可执行方案。

**详细说明**：
本智能体负责接收来自需求采集阶段的原始需求，进行深度分析、拆解和文档化。输出包括需求规格说明书（SRS）、需求追溯矩阵、边界分析报告等。不直接与客户沟通，只处理已采集的原始需求。
```

### 模块 2：核心能力（技能清单）

**目的**：列出这个智能体包含的所有技能，提供快速导航。

**内容要求**：
- 使用 Markdown 表格
- 表格列：技能名、说明、独立使用、文件链接
- 按技能的使用频率或逻辑顺序排列
- 每个技能链接到对应的技能文件

**示例**：

```markdown
## 核心能力（技能清单）

本智能体包含以下技能，可整体安装，也可按需单独取用：

| 技能 | 说明 | 独立使用 | 文件 |
|:---|:---|:---:|:---|
| 需求拆解 | 将高层需求分解为可实现的功能点 | ✅ | [需求拆解](skills/需求拆解.md) |
| 边界划分 | 明确系统边界，区分 scope 内/外 | ✅ | [边界划分](skills/边界划分.md) |
| 非功能需求提取 | 识别性能、安全、可用性等 NFR | ✅ | [非功能需求提取](skills/非功能需求提取.md) |
| SRS文档生成 | 生成软件需求规格说明书 | ✅ | [SRS文档生成](skills/SRS文档生成.md) |
| 追溯矩阵 | 建立需求-设计-测试追溯关系 | ❌ | [追溯矩阵](skills/追溯矩阵.md) |
```

**独立使用列说明**：
- `✅`：该技能可以脱离本智能体，单独使用
- `❌`：该技能依赖本智能体的上下文，不建议单独使用

### 模块 3：工作流说明

**目的**：说明这个智能体内部技能的执行顺序和协作关系。

**内容要求**：
- 用简单的流程图或步骤列表说明
- 标注哪些是串行（必须按顺序），哪些是并行（可选）
- 标注每个步骤的输入和输出

**示例**：

```markdown
## 工作流说明

```
原始需求输入
    ↓
[需求拆解] → 功能点列表
    ↓
[边界划分] → 系统边界图 + scope 清单
    ↓
[非功能需求提取] → NFR 清单（可与上一步并行）
    ↓
[SRS文档生成] → SRS 文档
    ↓
[追溯矩阵] → 需求追溯矩阵（依赖 SRS 输出）
```

**说明**：
- "需求拆解" → "边界划分" → "SRS文档生成" 建议按顺序串行执行
- "非功能需求提取"可在"需求拆解"后与"边界划分"并行执行
- "追溯矩阵"必须在"SRS文档生成"完成后执行
```

### 模块 4：安装说明

**目的**：告诉用户如何把这个智能体安装到自己的 AI 工具中。

**内容要求**：
- 分平台说明（Claude Code、Cursor、通用方式）
- 说明需要复制哪些文件
- 说明安装后的调用方式
- 如果安装复杂，提供分步指引

**示例**：

```markdown
## 安装说明

### 安装到 Claude Code

Claude Code 的 Skill 系统以单文件形式安装。将本智能体包含的各技能文件分别复制到 Skill 目录（全局 `~/.claude/skills/` 或项目级 `.claude/skills/`）：

| 源文件 | 安装路径 |
|:---|:---|
| `skills/需求拆解.md` | `~/.claude/skills/需求拆解.md` |
| `skills/边界划分.md` | `~/.claude/skills/边界划分.md` |
| `skills/非功能需求提取.md` | `~/.claude/skills/非功能需求提取.md` |
| `skills/SRS文档生成.md` | `~/.claude/skills/SRS文档生成.md` |
| `skills/追溯矩阵.md` | `~/.claude/skills/追溯矩阵.md` |

> **注意**：智能体定义文件（本文件）本身不安装到 Skill 目录，留在项目仓库中作为上下文参考。各技能 frontmatter 的 `name` 字段决定调用命令名（英文小写+连字符），详见 [05-多平台适配规范](05-多平台适配规范.md)。建议在智能体文件中建立中英对照表（如：需求分析 → requirement-analysis），便于团队成员快速定位。

### 安装到 Cursor

1. 将 `需求分析智能体.md` 的内容追加到项目根目录的 `.cursorrules` 文件中；
2. 或在 Cursor 对话中使用 `@` 引用该文件；
3. 各技能文件可在需要时单独 `@` 引用。

### 通用安装（任何 AI 工具）

1. 直接复制本文件内容，作为 system prompt 或上下文输入；
2. 需要具体技能时，打开对应的技能文件，复制其 Prompt 模板使用。
```

### 模块 5：使用示例

**目的**：展示一个完整的使用场景，帮助用户理解如何使用这个智能体。

**内容要求**：
- 1-2 个典型使用场景
- 每个场景包含：背景、操作步骤、预期输出
- 使用对话式或步骤式描述

**详细程度标准**：
- 每个场景至少包含：背景（1-2 句话）、3-5 个操作步骤、预期输出清单

**示例**：

```markdown
## 使用示例

### 示例 1：从原始需求到 SRS 文档

**背景**：产品经理小王已经通过客户访谈收集了 20 条原始需求记录，需要整理成正式的 SRS 文档。

**操作步骤**：
1. 打开 `需求拆解` 技能，输入 20 条原始需求
2. 获得功能点列表后，打开 `边界划分` 技能，确定系统边界
3. 并行执行 `非功能需求提取`，识别性能和安全要求
4. 将以上输出汇总，打开 `SRS文档生成` 技能
5. 获得初版 SRS 文档，人工评审后定稿

**预期输出**：
- 需求规格说明书（SRS）v1.0
- 系统边界图
- 需求追溯矩阵
```

### 模块 6：关联内容（可选）

**目的**：提供上下游和相关内容的导航。

**内容要求**：
- 上游：输入来自哪里（上一个智能体或人工产出）
- 下游：输出流向哪里（下一个智能体或人工处理）
- 相关：同集群或其他集群的相关智能体/技能

**跨集群链接**：智能体文件之间允许跨集群链接，使用相对路径 `../{序号}-{集群名}/{智能体文件}`，目标文件存在即可。如 `../02-设计工程/原型设计智能体.md`

**示例**：

```markdown
## 关联内容

### 上游输入
- [需求采集智能体](需求采集智能体.md) —— 提供原始需求

### 下游输出
- [数模生成智能体](数模生成智能体.md) —— 接收结构化需求进行建模
- [原型设计智能体](../02-设计工程/原型设计智能体.md) —— 接收需求进行原型设计

### 相关技能
- [需求去重](skills/需求去重.md) —— 如果原始需求有重复，先用此技能处理
```

## 六、禁止事项

1. **禁止在智能体文件中写技能的具体 Prompt**：智能体文件只放索引和说明，具体 Prompt 在技能文件中
2. **禁止放与技能无关的内容**：如通用编程规范、公司制度等，应放在单独的共享文档中
3. **禁止过长**：智能体文件应控制在 200 行以内（含示例和代码块），超限说明内容过多，需要精简
4. **禁止重复技能内容**：如果技能文件已详细说明，智能体文件中不要重复，只放链接
