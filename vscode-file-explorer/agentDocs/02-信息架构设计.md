# AI 技能知识库平台 —— 信息架构设计

## 一、设计原则

1. **角色优先**：目录结构让"我是谁"比"这是什么技术"更容易回答
2. **扁平可达**：从总索引到目标技能，最多 3 次点击/跳转
3. **自包含**：每个文件独立可读，不依赖外部上下文也能理解
4. **中文友好**：文件夹和文件主名使用中文，降低认知负担
5. **链接驱动**：通过 Markdown 相对链接建立关联，不依赖特殊工具

## 三、文件夹命名规范

### 3.1 集群文件夹

```
格式：{两位序号}-{中文集群名}

示例：
- 01-需求工程
- 02-设计工程
- 03-开发工程
```

**规则**：
- 序号固定两位，不足补零（01, 02, ..., 10）
- 序号表示推荐的浏览/学习顺序，也用于排序
- 中文名使用团队通用术语，不缩写

### 3.2 智能体文件

```
格式：{中文智能体名}智能体.md

示例：
- 需求采集智能体.md
- 原型设计智能体.md
- 前端开发智能体.md
```

**规则**：
- 文件名以"智能体"结尾，明确类型
- 使用中文，与文档中的称呼一致
- 一个智能体一个文件，不拆分

### 3.3 技能文件

```
格式：{中文技能名}.md

示例：
- 需求去重.md
- 根据需求生成原型.md
- React组件开发.md
```

**规则**：
- 文件名描述"做什么"，使用动词+名词结构
- 不使用"技能"后缀（已在 skills/ 目录下，无需重复）
- 文件名在全局唯一（即使不同集群也不重复）

## 四、索引系统设计

### 4.1 总索引 INDEX.md

**定位**：知识库唯一入口，所有用户首先看到的文件。

**必须包含的区块**：

1. **项目简介**（1-2 句话）
2. **按角色快速入门**（表格：角色 → 集群索引链接 → 说明）
3. **按场景快速查找**（表格：用户原话 → 技能直链）
4. **集群总览**（表格：序号 → 集群名 → 包含智能体数 → 包含技能数 → 索引链接）—— 共 6 大集群
5. **新手指引**（如何安装、如何使用）
6. **最近更新**（最近更新的 5 个技能/智能体）

**可选区块**：
- 使用统计/反馈入口
- 贡献指南链接

### 4.2 集群索引 clusters/{集群}/INDEX.md

**定位**：集群内部的导航中心。

**必须包含的区块**：

1. **集群简介**（1 段话说明这个集群解决什么问题）
2. **包含智能体**（表格：智能体名 → 定位 → 包含技能数 → 文件链接）
3. **全部技能列表**（表格：技能名 → 所属智能体 → 一句话说明 → 文件链接）
4. **常用技能推荐**（本集群最常被使用的 3-5 个技能）
5. **返回总索引**的链接

**排序规则**：
- 智能体按文件系统中的出现顺序
- 技能按"常用度"排序（初期按字母序，后期根据使用数据调整）

### 4.3 智能体技能清单

每个智能体文件内部的"技能清单"区块，也是索引的一部分：

```markdown
## 核心能力（技能清单）

| 技能 | 说明 | 独立使用 | 文件 |
|:---|:---|:---:|:---|
| 需求去重 | 识别并合并重复需求 | ✅ | [需求去重](skills/需求去重.md) |
| 需求拆解 | 将高层需求分解为可实现的功能点 | ✅ | [需求拆解](skills/需求拆解.md) |
```

**独立使用列**：标记该技能是否可以脱离智能体单独使用（✅/❌）。

## 五、导航系统设计

### 5.1 面包屑导航

每个文件顶部应有隐含的"面包屑"（通过链接实现）：

```markdown
<!-- 智能体文件顶部 -->
[← 返回需求工程集群索引](INDEX.md)
[← 返回总索引](../../INDEX.md)
```

### 5.2 关联导航

在文件底部提供相关内容的链接：

```markdown
## 关联内容

- [同集群其他智能体](INDEX.md)
- [相关技能：需求拆解](skills/需求拆解.md)
- [上级集群索引](INDEX.md)
```

### 5.3 搜索策略（无搜索功能时的替代方案）

由于没有 Web 搜索功能，通过以下方式辅助发现：

1. **总索引场景表**：覆盖用户最可能的搜索词
2. **技能文件标签**：frontmatter 中的 tags 字段，可被文档工具索引
3. **关键词重复**：在技能描述中重复用户可能搜索的词
4. **集群索引排序**：常用技能放在列表前面

## 六、链接规范

### 6.1 相对路径规则

| 源文件位置 | 目标文件位置 | 相对路径写法 |
|:---|:---|:---|
| `clusters/01-需求工程/INDEX.md` | `clusters/01-需求工程/需求分析智能体.md` | `[链接](需求分析智能体.md)` |
| `clusters/01-需求工程/需求分析智能体.md` | `clusters/01-需求工程/skills/需求拆解.md` | `[链接](skills/需求拆解.md)` |
| `clusters/01-需求工程/skills/需求拆解.md` | `clusters/01-需求工程/INDEX.md` | `[链接](../INDEX.md)` |
| `clusters/01-需求工程/INDEX.md` | `clusters/02-设计工程/INDEX.md` | `[链接](../02-设计工程/INDEX.md)` |
| 任何文件 | `INDEX.md`（根目录） | `[链接](../../INDEX.md)` |

**注意**：统一使用相对路径 `../../INDEX.md` 引用根目录总索引，不使用 `/INDEX.md` 的绝对路径写法。

**注意**：如果技能文件位于 `skills/子目录/` 下，到根目录总索引的路径应使用 `../../../INDEX.md`（相应增加 `../` 层级）。

### 6.2 链接失效检查

- 所有相对链接必须有效
- 使用 Markdown 链接检查工具（如 `markdown-link-check`）定期扫描
- 重命名文件时必须同步更新所有引用该文件的链接

## 七、元数据规范（Frontmatter）

### 7.1 所有文件通用字段

```yaml
---
name: "文件的唯一标识名（英文或中文，无空格，用-连接）"
description: "一句话描述这个文件的内容"
type: "index | agent | skill | template | redirect"
# 说明：模板文件的 type 值为 template
version: "1.0.0"
date: "2026-04-29"
---
```

### 7.2 智能体文件额外字段

```yaml
---
cluster: "需求工程"
roles: ["产品经理", "业务分析师"]
skills-count: 5
---
```

### 7.3 技能文件额外字段

```yaml
---
parent-agent: "需求分析智能体"
standalone: true
tags: ["需求", "分析", "文档"]
input: "原始需求列表"
output: "结构化的需求分析报告"
---
```

## 八、扩展性设计

### 8.1 新增集群

1. 在 `clusters/` 下新建文件夹 `{序号}-{集群名}/`
2. 创建 `INDEX.md`
3. 在根目录 `INDEX.md` 中添加该集群的入口
4. 除根目录 `INDEX.md` 外，无需修改其他现有文件

### 8.2 新增智能体

1. 在对应集群文件夹下新建 `{智能体名}智能体.md`
2. 在集群 `INDEX.md` 中添加该智能体的链接
3. 在总索引 `INDEX.md` 中更新该集群的智能体计数

### 8.3 新增技能

1. 在对应集群的 `skills/` 目录下新建 `{技能名}.md`
2. 在所属智能体文件中添加该技能到清单
3. 在集群 `INDEX.md` 中添加该技能的链接
4. 在总索引 `INDEX.md` 中更新该集群的技能计数（如有统计）

### 8.4 技能跨集群迁移

如果一个技能需要从一个集群移到另一个集群：
1. 移动文件到新位置
2. 更新所有引用该文件的链接
3. 在旧位置留下重定向说明（至少保留一个版本周期）

**重定向文件格式**：

在旧位置保留一个同名文件，内容只写：

```markdown
---
name: "需求去重"
description: "本技能已迁移"
type: "redirect"
version: "1.0.0"
date: "2026-04-29"
---

# 本技能已迁移

本技能/智能体已迁移至 [新位置](../新集群/skills/需求去重.md)，请前往查看。
```

**要求**：保留至少一个版本周期，确保外部链接和书签不会失效。

## 九、文件大小控制

| 文件类型 | 建议大小 | 最大大小 | 超限处理 |
|:---|:---|:---|:---|
| 总索引 INDEX.md | < 200 行 | 300 行 | 将"最近更新"和"新手指引"拆分到独立文件，总索引只留摘要和链接 |
| 集群索引 INDEX.md | < 150 行 | 200 行 | 按智能体分区块折叠（每个智能体一个折叠区块） |
| 智能体定义文件 | < 150 行 | 200 行 | 将详细说明拆到单独文档 |
| 技能文件 | < 150 行 | 200 行 | 拆分为核心版+扩展版 |

**目的**：保证每个文件在屏幕内可快速浏览，降低认知负担。
