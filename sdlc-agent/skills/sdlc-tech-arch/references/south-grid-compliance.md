# 南网 / 电力行业合规检查清单

## 使用说明
此清单在 **需求评审阶段** 和 **技术架构阶段** 必须逐项检查。对于 South Grid 项目，以下合规项必须确认。

## 1. 等级保护（等保）

### 检查项
- [ ] 系统是否需要通过等保测评？
- [ ] 等保级别：一级 / 二级 / **三级**（电力行业通常为三级）/ 四级？
- [ ] 等保测评报告是否已有？是否需要新增？

### 等保三级要求（电力行业常见）
| 维度 | 要求 | 技术实现 |
|------|------|---------|
| 身份鉴别 | 双因素认证 | 密码 + 短信/令牌 |
| 访问控制 | 最小权限原则 | RBAC + 数据权限 |
| 安全审计 | 操作日志完整保留 | 操作日志表，保留 ≥ 6 个月 |
| 数据完整性 | 传输和存储加密 | HTTPS + 数据库字段加密 |
| 数据保密性 | 敏感数据加密 | AES 加密存储 |
| 备份恢复 | 定期备份 + 演练 | 数据库备份 + 容灾方案 |

## 2. 国产化 / 信创要求

### 检查项
- [ ] 是否有国产化要求？
- [ ] 需要国产化的组件：
  - [ ] 操作系统（麒麟/统信/欧拉）
  - [ ] 数据库（达梦/人大金仓/华为 GaussDB）
  - [ ] 中间件（东方通/TongWeb）
  - [ ] 浏览器（360/奇安信/统信）
  - [ ] 服务器芯片（鲲鹏/飞腾/海光）

### 国产化适配指南
| 层级 | 推荐选择 | 适配注意 |
|------|---------|---------|
| 操作系统 | 麒麟 V10 / 统信 UOS | 文件路径、系统命令差异 |
| 数据库 | 达梦 DM8 / 华为 DWS | SQL 语法差异，见兼容性文档 |
| 中间件 | 东方通 TongWeb 7+ | Servlet 容器配置差异 |
| 浏览器 | 360 安全浏览器 / 奇安信 | ES6 支持度、WebSocket 限制 |

## 3. 数据安全

### 检查项
- [ ] 敏感数据是否加密存储？（用户电费、手机号、地址）
- [ ] 传输是否使用 HTTPS？
- [ ] 数据权限是否按组织层级隔离？（省/地市/区县/班组）
- [ ] 数据导出是否有审批和审计？
- [ ] 数据库密码是否加密配置？
- [ ] 是否有数据脱敏机制？（开发/测试环境）

### 数据脱敏规则
| 数据类型 | 脱敏规则 | 示例 |
|---------|---------|------|
| 手机号 | 中间 4 位隐藏 | 138****1234 |
| 身份证号 | 中间 10 位隐藏 | 110**********1234 |
| 地址 | 保留到区县 | 广东省广州市天河区*** |
| 电费金额 | 保留整数 | 123.45 → 123 |
| 用户姓名 | 保留姓氏 | 张三 → 张* |

## 4. 电力行业特定规范

### 检查项
- [ ] 是否符合《电力行业信息安全等级保护管理办法》？
- [ ] 是否符合《电力监控系统安全防护规定》（发改委 14 号令）？
- [ ] 生产控制大区与管理信息大区的隔离要求？
- [ ] 系统是否属于生产控制大区？如果是，需要额外的安全隔离措施。

### 大区分隔
| 大区 | 系统类型 | 安全要求 |
|------|---------|---------|
| 生产控制大区 | SCADA、用电采集、负控 | 物理隔离，禁止外网访问 |
| 管理信息大区 | 营销系统、OA、报表 | 逻辑隔离，允许内网访问 |
| 互联网大区 | 对外服务 APP、公众号 | 边界防护，通过网闸/防火墙 |

## 5. 审计与日志

### 检查项
- [ ] 所有增删改操作是否记录审计日志？
- [ ] 审计日志包含：用户、时间、IP、操作类型、操作对象、变更前后值？
- [ ] 日志保留时间：≥ 6 个月（等保三级要求）
- [ ] 日志是否防篡改？（只写权限、日志服务器分离）
- [ ] 是否有登录失败锁定机制？（连续 5 次失败锁定 30 分钟）

### 审计日志字段
```sql
CREATE TABLE sys_audit_log (
    id BIGINT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    username VARCHAR(50) NOT NULL,
    operation VARCHAR(100) NOT NULL,      -- 操作类型
    method VARCHAR(200) NOT NULL,          -- 请求方法
    params TEXT,                            -- 请求参数
    ip VARCHAR(50) NOT NULL,               -- 客户端 IP
    duration INT,                           -- 执行耗时(ms)
    status INT,                            -- 操作结果(0=成功, 1=失败)
    error_msg TEXT,                        -- 错误信息
    create_time TIMESTAMP NOT NULL         -- 操作时间
);
```
