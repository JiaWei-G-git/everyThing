# 测试用例模板

## 使用说明
所有测试用例使用此模板编写，确保覆盖功能、API、权限三个维度。

## 模板格式

```markdown
### TC-{SEQ}: [功能名称] - [测试场景]

- **测试用例 ID**: TC-{SEQ}
- **关联需求**: REQ-{MODULE}-{NNN}
- **关联 API**: API-{NNN}（如适用）
- **测试类型**: 功能测试 / API 测试 / 权限测试 / E2E 测试
- **优先级**: P0（阻塞） / P1（重要） / P2（一般）
- **所属模块**: [模块名]

#### 前置条件
1. [前置条件1]
2. [前置条件2]

#### 测试数据
| 字段 | 值 | 说明 |
|------|------|------|
| [字段1] | [值1] | [说明] |
| [字段2] | [值2] | [说明] |

#### 测试步骤
1. [步骤1]
2. [步骤2]
3. [步骤3]

#### 预期结果
- [结果1]
- [结果2]

#### 实际结果（执行后填写）
- [ ] 通过
- [ ] 失败（备注：______）

#### 截图/附件
- [附件链接]
```

## 测试类型说明

| 类型 | 覆盖范围 | 执行方式 |
|------|---------|---------|
| 功能测试 | 业务功能正确性 | 手工 / 自动化 |
| API 测试 | 接口请求/响应/参数验证 | 自动化（JUnit/pytest） |
| 权限测试 | 角色权限控制 | 自动化 + 手工 |
| E2E 测试 | 端到端用户流程 | 自动化（Playwright/Cypress） |
| 性能测试 | 响应时间/并发 | 工具（JMeter/k6） |
| 兼容性测试 | 浏览器/设备兼容性 | 手工 |

## 优先级定义

| 优先级 | 定义 | 覆盖标准 |
|--------|------|---------|
| P0 | 阻塞性 | 核心业务流程，必须 100% 覆盖 |
| P1 | 重要 | 主要功能，覆盖 ≥ 80% |
| P2 | 一般 | 次要功能，覆盖 ≥ 50% |

## 测试用例生成规则

### 每个功能点至少生成 3 个用例

1. **正向用例**：正常输入，预期成功
2. **异常用例**：无效输入，预期错误处理
3. **边界用例**：边界值（空值、最大值、最小值、超长字符串）

### 每个 API 端点至少生成 4 个用例

1. 正常请求 → 200 成功
2. 缺少必填参数 → 400 参数错误
3. 无权限访问 → 403 禁止访问
4. 资源不存在 → 404 未找到

### 每个角色权限组合至少生成 1 个用例

遍历权限矩阵中的每个角色 + 页面 + 操作组合，验证允许/禁止。

## 示例

```markdown
### TC-001: 用户管理 - 新增用户成功

- **测试用例 ID**: TC-001
- **关联需求**: REQ-USER-001
- **关联 API**: API-001
- **测试类型**: 功能测试 + API 测试
- **优先级**: P0
- **所属模块**: 用户管理

#### 前置条件
1. 管理员已登录
2. 待分配的角色已存在

#### 测试数据
| 字段 | 值 | 说明 |
|------|------|------|
| username | test_user_001 | 唯一用户名 |
| real_name | 测试用户 | 真实姓名 |
| role_id | 2 | 普通用户角色 |
| org_id | 3 | 天河区 |
| phone | 13800138000 | 手机号 |
| password | Test@123456 | 符合密码规则 |

#### 测试步骤
1. 进入用户管理页面
2. 点击"新增用户"按钮
3. 填写表单所有必填字段
4. 选择角色"普通用户"
5. 选择组织"天河区"
6. 点击"保存"

#### 预期结果
- 页面提示"新增成功"
- 用户列表中出现新增用户
- 数据库 sys_user 表新增一条记录
- 数据库 sys_user_role 表新增关联记录
- API 返回 { code: 200, message: "success", data: { id: 1001 } }

#### 实际结果
- [ ] 通过
- [ ] 失败

---

### TC-002: 用户管理 - 新增用户（用户名已存在）

- **测试用例 ID**: TC-002
- **关联需求**: REQ-USER-001
- **测试类型**: 功能测试 + API 测试
- **优先级**: P1

#### 测试步骤
1. 进入用户管理页面
2. 点击"新增用户"
3. 填写已存在的用户名
4. 点击"保存"

#### 预期结果
- 页面提示"用户名已存在"
- API 返回 { code: 400, message: "用户名已存在" }
- 数据库无新增记录

---

### TC-003: 用户管理 - 普通用户无权限删除用户

- **测试用例 ID**: TC-003
- **关联需求**: REQ-USER-004
- **测试类型**: 权限测试
- **优先级**: P0

#### 测试步骤
1. 使用普通用户账号登录
2. 进入用户管理页面
3. 尝试点击删除按钮

#### 预期结果
- 删除按钮不可见（前端权限控制）
- 或直接点击后返回 403 无权限
- 数据库无删除操作
```
