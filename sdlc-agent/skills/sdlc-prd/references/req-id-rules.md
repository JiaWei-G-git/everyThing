# REQ-ID 命名规范

## 格式
```
REQ-{MODULE}-{SEQ}
```

## 模块缩写对照表

| 模块 | 缩写 | 说明 |
|------|------|------|
| 用户管理 | USER | 用户、账号、登录 |
| 角色权限 | ROLE | 角色、权限、菜单 |
| 组织机构 | ORG | 省/地市/区县/班组/供电所 |
| 系统管理 | SYS | 系统配置、日志、字典 |
| 计量管理 | METER | 电能表、计量点、采集 |
| 台区管理 | TAI | 台区、变压器、拓扑 |
| 线损分析 | LOSS | 线损计算、异常分析 |
| 数据采集 | COL | 采集终端、通信状态 |
| 电能量 | ENERGY | 电量、结算、统计 |
| 数据报表 | REPORT | 报表、图表、导出 |
| 异常预警 | ALARM | 告警、预警、处理 |
| 运维工单 | WORK | 工单、任务、巡检 |
| 通用后台 | ADMIN | 通用后台管理功能 |

## 序号规则
- 从 `001` 开始，三位数字，不足补零
- 同一模块内连续递增，不跳号
- 删除需求时不回收序号，保持历史连续性
- 新增需求时取当前最大序号 + 1

## 示例
```
REQ-USER-001: 用户登录
REQ-USER-002: 用户新增
REQ-USER-003: 用户编辑
REQ-USER-004: 用户删除
REQ-USER-005: 用户密码重置

REQ-ROLE-001: 角色管理
REQ-ROLE-002: 权限分配
REQ-ROLE-003: 菜单管理

REQ-METER-001: 电能表档案管理
REQ-METER-002: 计量点管理
REQ-METER-003: 采集终端管理
```

## 扩展规则
当模块缩写不够用时：
1. 新增缩写到对照表
2. 缩写长度不超过 5 个字符
3. 使用大写字母，避免与已有缩写冲突
