## 命名规范

### REQ-ID 格式
- 格式：`REQ-{MODULE}-{SEQ}`
- 模块缩写：USER（用户）、ROLE（角色）、METER（计量）、DATA（数据）、REPORT（报表）、SYS（系统）
- 示例：`REQ-USER-001`、`REQ-METER-003`
- 规则：
  - 模块缩写使用大写字母，不超过 5 个字符
  - 序号从 001 开始，连续递增
  - 同一模块的需求顺序排列

### API ID 格式
- 格式：`API-{SEQ}`
- 示例：`API-001`、`API-002`
- 规则：全项目统一递增，不按模块分组

### 页面命名
- 列表页：`[实体]列表` 或 `[实体]管理`
- 表单页：`[实体]新增` 或 `[实体]编辑`
- 详情页：`[实体]详情`
- 路由：`/[模块]/[实体]` 或 `/[模块]/[实体]/[操作]`
