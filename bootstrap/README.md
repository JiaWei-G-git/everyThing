## 🎯 给 AI 的生成指令

你是一个 AI 编码助手，负责根据规范文档自动生成完整的项目文件结构。当前工作目录即为项目根目录，所有规范文档（编号 01 至 07 的 `.md` 文件）均位于本目录下。

**执行流程**：

1. 扫描当前目录下所有 `.md` 文件，逐一读取其完整内容，建立对项目整体结构、文件清单、内容要求的全局理解
2. 严格按照文件名编号顺序（01 → 02 → … → 07）依次处理每份规范文档中定义的生成任务
3. 对每份规范文档，先解析其中声明的目录结构，使用 `mkdir -p` 逻辑递归创建所有必要的目录层级，再逐一写入该规范要求的每个文件
4. 每个文件写入完成后立即继续处理下一个文件，无需等待用户确认或反馈
5. 全部文件生成完毕后，输出一份完整的文件清单报告，格式为树形目录结构，标注每个文件对应的来源规范编号（如 `[来源: 03-xxx.md]`）

**路径与编码约束**：

- 所有文件路径和目录路径必须使用相对于项目根目录的相对路径，禁止使用绝对路径（如 `/home/...` 或 `~/...`）
- 目录名和文件名中包含中文字符的，保持原始中文不变，禁止转译为拼音、英文或 URL 编码
- 所有生成的 Markdown 文件（`.md`）必须使用 UTF-8 编码，文件开头不添加 BOM 标记

**异常处理**：

- 若规范文档之间存在路径冲突或内容矛盾，以编号较大的文档为准
- 若某份规范文档中引用了其他规范文档中定义的目录或文件，确保按依赖顺序先创建被引用的资源
- 若规范文档中某个文件的内容描述不完整或存在歧义，基于上下文和项目整体意图进行合理推断并生成，同时在最终报告中标注 `[推断生成]`
