# 知识库架构与流转流程

> 精简版架构参考。完整实现见各 Skill 定义和模板文件。

## 双轴驱动模型

```
原料层(输入)                      资产层(产出)
┌─────────────────────┐         ┌──────────────────────┐
│ 00-Inbox   快速捕获  │  ───▶  │ 10-Prompts  提示词    │
│ 01-Work    工作实践  │  ───▶  │ 20-Agents   代理      │
│                      │         │ 30-Skills   技能      │
│ (你往里扔东西)        │         │ 40-MCP      服务      │
│                      │         │ 50-Workflows 工作流   │
│                      │         │ 60-Tutorials 教程     │
│                      │         │ 70-Sharing  团队共享  │
└─────────────────────┘         └──────────────────────┘
```

## 三条流转路径

```
单次原料（外部教程/灵感）
      │
      └─── AI ──▶ asset-extractor ──▶ 00-Inbox/待提炼
                                        ↓
                                   人工确认
                                        ↓
                      ┌─── Prompt   → 10-Prompts
                      ├─── Agent    → 20-Agents
多次会话（≥3次同类）   ├─── Skill    → 30-Skills
      │               ├─── MCP      → 40-MCP
      └─── AI ──▶ pattern-miner ──▶ 50-Workflows
 跨案例抽象）           └─── Tutorial → 60-Tutorials

成熟资产 ──▶ 70-Sharing(团队分发)
失效内容 ──▶ 99-Archive
```

## 五条核心规则

1. **Inbox 只是中转站**：外部教程、临时想法先扔 00-Inbox，每周归纳一次，空掉它。
2. **Work 按项目聚合**：每个项目一个子目录，内部分 `会话记录/日志/文档`。
3. **会话记录是纯日志**：单次会话不具备通用性，保存后直接归档，等待周期性盘点（≥3 次才提炼）。
4. **资产来源可追溯**：每条资产标注来源原料，跨案例抽象时标注来源会话数量。
5. **原料不删除**：提炼后原料保留，仅打 `已提炼` 或 `pattern_extracted` 标签。

## 迭代节奏

| 频率 | 动作 | 目标 |
|------|------|------|
| **日常** | 使用 AI → 触发 `保存会话`（5 分钟） | 原料不流失 |
| **每周** | 触发 `盘点本周模式`(pattern-miner) | 提炼有质量 |
| **每周** | 整理 00-Inbox，提炼外部教程 | Inbox 清零 |
| **每月** | 评估闭环、审阅 `待提炼/` 积压 | 避免僵化 |
