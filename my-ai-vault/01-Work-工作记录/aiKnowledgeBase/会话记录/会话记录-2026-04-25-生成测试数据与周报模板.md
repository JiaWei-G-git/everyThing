# 会话记录 - 生成测试数据与周报模板

## 背景

项目框架搭建完成后，需要准备测试数据来验证功能闭环：会话记录保存、资产提炼、周报生成。需要多个项目类型和多个周报模板才能充分测试。

## 用户提示词序列

```
[1] 生成一些测试数据，包括黑客松项目的会话记录以及收件箱待整理的内容，包括会话记录、教程文档等内容

[2] 生成几个周报模板以及不同项目的会话记录，要来测试不同模板生成的周报内容

[3] 保存会话记录（第一次尝试，未严格按 Skill 规范执行）

[4] 那删除刚才生成的文件

[5] save-session（第二次，按规范执行，但提取逻辑有缺陷）

[6] 这些应该是自动采集的吧，不应该我来输入

[7] 确认

[8] 我的上下文提示词怎么只剩下了 save-session 这一个

[9] 不应该只提取一次的提示词，是要整个会话中所有的提示词才对

[10] 现在修改保存会话 skill，需要判断出哪些提示词是实际解决问题的，哪些是无效的询问，只提取出有效提示词

[11] 然后采集会话信息这部分不应该像用户收集，应该是从会话上下文中收集
```

## 解决思路

**测试数据设计**：

| 项目 | 类型 | 会话记录 |
|------|------|---------|
| 订单系统重构 | 后端 | 分库分表 / Redis 缓存 / 接口压测 |
| 组件库建设 | 前端 | Button 规范 / 主题切换 / 文档站点 |
| 客户 A 系统对接 | 客户交付 | 需求评审 / 联调排查 / 上线验收 |
| 黑客松项目 | 内部 | 架构设计 / Skill 定义 / 项目收尾 |

**周报模板设计**：
- 默认模板：通用 5 维度（进展/成果/问题/资产/计划）
- 敏捷开发模板：增加 Sprint 目标、进行中事项、技术债务、团队反馈
- 客户项目模板：增加交付物、客户沟通、需求变更、里程碑、待确认事项
- 技术调研模板：增加调研目标、方案对比、实验结果、结论建议、待决策

**Skill 迭代**：
- 第一次 save-session：AI 自动代劳，未交互式确认，被批评
- 第二次 save-session：按规范执行，但只提取了 `save-session` 单条提示词
- 第三次改进：改为"用户提示词序列"，提取所有消息
- 第四次改进：增加有效提示词筛选，只保留实质性任务

## 优化经验

1. 测试数据的会话记录最好使用真实的日期（本周内），这样测试周报生成时"最近7天"的过滤逻辑能正常生效
2. 不同项目类型的会话记录内容差异要大，才能测试出模板选择逻辑是否正确
3. **Skill 的迭代过程本身就是最好的测试数据**。从"自动代劳"到"规范执行"到"有效筛选"，每一步的改进都反映了实际使用中的痛点
4. 客户项目的记录要包含"需求变更""客户反馈"等维度，才能验证客户项目模板的特殊字段
5. 后端项目的记录要包含"阻塞项""技术债务"，才能验证敏捷模板的对应字段
