# 会话记录 - 测试会话记录保存 Skill

## 背景

验证 AI 知识库智能体的会话记录保存功能是否严格按照 Skill 规范执行。之前自动保存时被指出未遵循交互式采集流程，本次重新按规范触发。

## 用户提示词序列

按时间顺序列出本次会话中**有效**的用户提示词（已过滤掉触发词、追问、操作指令和纯反馈）：

```
[1] 生成一些测试数据，包括黑客松项目的会话记录以及收件箱待整理的内容，包括会话记录、教程文档等内容

[2] 生成几个周报模板以及不同项目的会话记录，要来测试不同模板生成的周报内容
```

**被过滤掉的无效提示词**：
- "保存会话记录" / "save-session" → 触发词，无实质内容
- "这次保存是否使用了skill"（×3）→ 追问，非任务指令
- "那删除刚才生成的文件" → 操作指令，纠正行为
- "这些应该是自动采集的吧" → 反馈讨论，无新任务
- "确认" → 纯确认
- "我的上下文提示词怎么只剩下了 save-session" → 反馈讨论
- "不应该只提取一次的提示词" → 反馈讨论

## 解决思路

按 session-recorder SKILL.md 的 6 步流程执行：

1. 读取项目级配置（AGENTS.md）
2. 确认路径有效（01-Work-工作记录/黑客松项目/会话记录）
3. 复用 session_archive_path
4. 从当前对话上下文中自动采集会话信息
5. 渲染标准模板
6. 保存文件并通知用户

关键改进：
1. 本次执行改为"自动提取 + 用户确认"模式，而非之前的"AI 脑补 + 直接写入"
2. 将"核心 Prompt"提取逻辑从"单条提取"修正为"用户提示词序列"——按时间顺序提取会话中所有用户发出的实质性提示词/指令
3. 增加**有效提示词筛选机制**：区分"实质性任务"与"无效询问"，只保留真正推动问题解决的用户指令
4. 筛选规则：触发词、追问、操作指令、纯确认、纯反馈——全部过滤掉

## 优化经验

1. Skill 触发后应优先自动提取上下文，仅对不确定字段询问用户，减少用户输入负担
2. 来源字段可由 AI 工具自行识别，无需用户输入
3. 项目名从已配置的 session_archive_path 推断比询问更高效
4. 自动提取后向用户展示摘要并请求确认，既保证效率又确保准确性
5. 用户提示词序列比单条 Prompt 更能反映会话的完整脉络，后续回顾时能看到问题如何演进
6. 有效提示词筛选是必需的：一个保存会话的触发过程可能包含10+条消息，但真正的实质任务可能只有2-3条，过滤噪声能大幅提升归档质量
