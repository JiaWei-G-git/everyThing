# 会话记录 - 重新扫描Vault并同步更新项目文件

## 背景

用户在日常维护 AI 知识库项目过程中，需要对 Vault 实际目录结构与项目文档进行同步更新，包括：重新扫描 my-ai-vault 文件夹、修正文档中的目录数量与结构描述、更新资产盘点、调整 Git 跟踪规则、将根目录 templates/ 迁移至 Vault 内并去掉中文目录名。

## 用户提示词序列

按时间顺序列出本次会话中用户发出的所有实质性提示词/指令：

```
1. @my-ai-vault 重新扫描文件夹 其他项目文件进行同步更新
2. @templates 这个文件夹是做什么用的
3. 这个模板可以放在 @my-ai-vault\90-Templates-模板 这里面把？
4. vault 里面的内容也要添加到git中
5. 把模板内容放到 @my-ai-vault\90-Templates-模板 中 并且把这个文件夹的中文去掉
6. 记录本次回话save-session
```

## 解决思路

1. 使用 PowerShell 扫描 my-ai-vault 实际目录与文件结构，解决中文编码问题（`[Console]::OutputEncoding = [System.Text.Encoding]::UTF8`）
2. 对比 PROJECT_GENERATOR.md、README.md、PROJECT.md、AGENTS.md、docs/知识库架构与流转流程-2026-04-25.md 与实际结构的差异
3. 批量执行最小化修改：补全 00-Inbox 未分类子目录、填充资产盘点、修正目录数量、移除 F-009"规划中"标记、调整 Git 忽略规则
4. 执行文件系统操作：重命名 90-Templates-模板 → 90-Templates，移动根目录 templates/ 文件，删除空目录
5. 全库文本替换：更新所有 SKILL.md 和项目文档中的模板路径与目录结构描述

## 优化经验

1. Windows PowerShell 默认输出中文为乱码，扫描前必须显式设置 `[Console]::OutputEncoding = [System.Text.Encoding]::UTF8`
2. PowerShell 不支持 `&&` / `||` 语法，条件判断需用 `if (...) { } else { }`
3. 批量替换 `templates/` 时需区分根目录模板路径与普通英文单词"templates"，避免误替换
4. 对于大型蓝图文件 PROJECT_GENERATOR.md，建议先 grep 定位再分段读取，避免上下文溢出
5. 执行文件移动/重命名前，先用 todo list 拆分任务，降低操作遗漏风险
