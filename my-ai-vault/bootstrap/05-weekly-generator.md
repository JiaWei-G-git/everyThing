# 周报生成技能 (Weekly Generator)

**版本**: v1.0.0
**类型**: Skill · 项目文档自动生成器

当用户说 **"生成本周周报"**、**"写周报"** 或表达类似意图时，AI 自动匹配本 Skill。

> 💡 **手动调用**：`/skill:weekly-generator`

---

## 功能说明

按项目聚合 `01-Work-工作记录/` 下的会话记录与日志，调用 LLM 生成符合模板的周报或项目情况文档。支持自定义周报模板。

---

## 执行步骤

### 1. 读取项目级配置

检查当前工作目录下的 `AGENTS.md` / `CLAUDE.md`，读取：
- `session_archive_path` — 确定项目归档根目录
- `weekly_template` — 自定义周报模板路径（可选）

```markdown
<!-- AI-Vault-Config -->
- session_archive_path: "01-Work-工作记录/XX项目/会话记录/"
- weekly_template: "90-Templates/周报/敏捷开发-周报模板.md"
```

### 2. 确定项目与周期

**交互式确认**：
- 项目名（从路径中推断或询问用户）
- 周期：日 / 周 / 月（默认周）
- 时间范围（默认最近 7 天）

### 3. 聚合原料内容

**时间筛选规则（关键）**：
- **筛选基准**：会话记录 YAML Frontmatter 中的 `date` 字段，**绝不使用文件修改时间**
- **无 `date` 字段的文件**：尝试从文件名提取日期（如 `会话记录-2026-04-25-xxx.md`），提取失败则归入"未分类历史记录"
- **时间范围外的文件**：仅作为历史上下文参考，不纳入本周统计
- **工作日志**：同理按日志文件中的 `date` 字段筛选

扫描 `01-Work-工作记录/<项目名>/` 下的内容：
- 会话记录（`会话记录/`）
- 工作日志（`日志/`）
- 已有周报（`文档/` 下的历史周报，用于增量参考）

按以下维度聚合：
- 完成事项（Completed）
- 进行中事项（In Progress）
- 阻塞项与风险（Blockers）
- 关键决策与方案（Key Decisions）
- 下阶段计划（Next Plan）
- 资产沉淀清单（本周提炼了哪些新资产）

#### 3.1 智能去重

同一事项可能在多个会话中重复记录。聚合时执行去重，避免周报中重复列出：

**去重规则**：
1. 比较同一维度下各条目的 `topic` 和核心内容
2. 若描述的是**同一任务的不同阶段**（如"设计接口"→"实现接口"→"测试接口"）→ 合并为一条，标注进度演变
3. 若描述的是**完全相同的进展**（如两次会话都提到"已完成用户模块"）→ 合并为一条，去重
4. 若描述的是**独立任务**（如"接口设计"和"数据库迁移"）→ 保留为独立条目

**合并格式**：
```
- 完成用户认证模块开发（涉及 3 条会话）
  - 4/21: 完成接口设计 → 4/23: 完成编码实现 → 4/24: 通过单元测试
```

### 4. 选择周报模板

**模板优先级**：
1. 项目级配置中的 `weekly_template`
2. `90-Templates/周报/<项目类型>-周报模板.md`
3. `90-Templates/周报/默认周报模板.md`

**模板存放规范**：
```
90-Templates/周报/
├── 默认周报模板.md
├── 敏捷开发-周报模板.md
├── 客户项目-周报模板.md
└── 技术调研-周报模板.md
```

**模板语法**：Jinja2 占位符
```markdown
## 本周进展
{{本周进展}}

## 阻塞项
{{阻塞项}}

## 资产沉淀
{{资产沉淀}}
```

### 5. 调用 LLM 生成文档

将聚合内容 + 模板发送给 LLM，生成结构化 Markdown 文档。

**LLM 系统提示词**：
```
你是一位技术项目经理。请根据以下项目原料内容，生成一份结构化的周报。

原料内容包括：
- 会话记录（含背景、解决思路、优化经验）
- 工作日志
- 历史周报（用于增量参考）

请按模板中的占位符填充内容，遵循以下原则：
1. 本周进展：总结已完成的事项，用 bullet points
2. 关键成果：提炼有产出的成果，尽量量化
3. 问题与解决方案：列出遇到的问题及已采取的解决措施
4. 资产沉淀：列出本周新提炼或确认的知识资产（Prompt、Skill、文档等）
5. 下周计划：基于当前进展和阻塞项，给出合理的下一步计划
6. 如无内容则不编造：某维度无素材时如实写"本周无相关记录"

输出格式：标准 Markdown，保持模板原有结构。
```

**LLM 调用参数**：
- temperature: 0.5（平衡创造性和一致性）
- 超时: 120 秒（聚合内容可能较长）

### 6. 质量自检（基础）

生成后执行基础检查：

| 检查项 | 规则 | 失败处理 |
|--------|------|----------|
| 关键字段非空 | `本周进展`、`下周计划` 不能为空 | 若为空，补充"本周无相关记录" |
| 阻塞项闭环 | 每个 `Blocker` 应有对应 `解决方案` 或 `状态说明` | 若无，提示用户补充 |
| 资产链接有效性 | `[[WikiLink]]` 格式的链接指向的文件应在 Vault 中存在 | 标记为 `[链接待确认]` |
| 日期一致性 | 周报日期应在请求的时间范围内 | 超出范围的内容归入"历史参考" |
| 重复内容 | 与历史周报对比，避免完全相同的条目 | 合并为"持续推进" |

**质量检查不通过时**：向用户汇报问题清单，询问是否继续保存或补充信息。

### 7. 保存并通知

**输出路径**：
```
01-Work-工作记录/<项目名>/文档/<项目名>-周报-YYYY-MM-DD.md
01-Work-工作记录/<项目名>/文档/项目情况-<项目名>.md（持续追加）
```

**文件生成规则**：
- 周报文件：每次生成独立文件，文件名格式 `{项目名}-周报-{YYYY-MM-DD}.md`
- 项目情况文档：若已存在，在文件末尾追加本次生成的内容（前加 `---` 分隔线）；若不存在则新建
- 去重：若当日报周文件已存在，追加序号（如 `-1`）

向用户汇报生成结果和文件路径。

---

## 核心规则（不可违反）

1. **模板可自定义** — 项目可在 `AGENTS.md` 中指定专属模板，未指定则使用默认模板；
2. **项目情况文档持续追加** — 非覆盖，每次生成追加到现有文件末尾；
3. **周报独立文件** — 每次生成独立文件，按日期命名；
4. **自定义维度支持** — 项目可在配置中定义额外聚合维度（如"客户需求变更"、"上线风险"）；
5. **无内容不硬编** — 如某周期内无会话记录，如实汇报"本周无记录"，不编造内容。

## 使用示例

**用户**: "生成本周周报"

**执行**:
1. 读取项目级配置，确定项目名和模板；
2. 扫描最近 7 天的会话记录和日志；
3. 按维度聚合内容；
4. 调用 LLM 生成符合模板的周报；
5. 执行基础质量检查；
6. 保存到 `01-Work-工作记录/<项目名>/文档/`；
7. 通知用户。
