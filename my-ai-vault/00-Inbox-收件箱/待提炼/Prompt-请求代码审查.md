# 请求代码审查

## 场景
在子代理驱动开发中的每个任务完成后、完成主要功能后、或合并到主分支前，对照计划审查代码质量与规范符合性。

## 模板

请对以下代码变更进行审查，对照原始计划检查是否符合要求。

**原始计划**：
{{实现计划文档或关键要求摘要}}

**变更范围**：
- BASE_SHA: {{起始提交哈希}}
- HEAD_SHA: {{最新提交哈希}}

**审查重点**：
1. **规范符合性**：代码是否实现了计划中描述的功能？有无遗漏或偏离？
2. **代码质量**：命名是否清晰？逻辑是否简洁？有无重复代码？
3. **测试覆盖**：是否包含验证该功能的测试？测试是否有效？
4. **边界处理**：是否处理了错误情况和边界条件？
5. **安全与性能**：有无明显的安全隐患或性能问题？

**输出要求**：
- 按严重性分类问题：严重（必须立即修复）、重要（继续前修复）、建议（可选改进）
- 对每个问题提供具体位置和改进建议
- 如果代码完全符合要求，明确声明"审查通过"

**审查原则**：
- 基于技术事实，不附和、不猜测
- 优先关注正确性和可维护性
- 区分风格偏好和真正的问题
