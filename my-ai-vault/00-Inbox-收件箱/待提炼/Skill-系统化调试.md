# 系统化调试

## 触发条件
在遇到任何错误、测试失败或意外行为时，**在提出修复之前**使用。

## 执行步骤

1. **根本原因调查**
   - 完整阅读错误消息和堆栈跟踪
   - 尝试一致地重现问题（最小化复现步骤）
   - 检查最近的代码变更、配置更改或依赖更新
   - 收集所有相关证据（日志、输入数据、环境信息）
   - 预期结果：明确的问题复现路径和初步证据清单

2. **模式分析**
   - 找到系统中正常工作的相似功能或历史版本
   - 对比正常工作与失败场景的差异
   - 识别导致失败的特定条件或输入
   - 预期结果：缩小问题范围到具体代码区域或逻辑分支

3. **假设和测试**
   - 基于证据形成单一、具体的失败假设
   - 设计最小化测试来验证该假设
   - 运行验证测试，确认假设成立或排除
   - 预期结果：确定的根本原因解释

4. **实施修复**
   - 创建失败测试用例（如尚未存在），确保能捕获该 bug
   - 实施单一、最小的修复，直接针对根本原因
   - 运行完整测试套件验证修复不引入新问题
   - 预期结果：测试全部通过，问题被根本性解决

## 铁律
```
没有根本原因调查，就没有修复
```
