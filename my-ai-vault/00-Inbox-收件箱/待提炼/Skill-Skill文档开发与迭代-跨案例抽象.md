# Skill 文档开发与迭代

> 基于 5 条会话记录跨案例抽象（创建 → 编写 → 测试 → 修改 → 批量优化）

## 触发条件
需要为 AI 工具创建新的 Skill 定义文档，或基于使用反馈对现有 Skill 进行迭代优化时。

## 输入 / 输出

| 类型 | 说明 |
|------|------|
| **输入** | 用户自然语言指令、使用过程中的反馈、问题分析报告、Brainstorming 分析文档 |
| **输出** | `SKILL.md` 文件、配套模板文件、同步更新的项目文档（如 bootstrap/） |

## 通用步骤（基于 ≥3 条记录归纳）

### 1. 定义文档标准结构
为所有 Skill 统一以下章节顺序，确保 AI 工具间的一致性：
```
# Skill 名称
版本 + 类型 + 触发条件
---
功能说明
---
执行步骤（1. 2. 3. ...）
---
核心规则（不可违反）
---
使用示例
---
版本信息
```

### 2. 编写触发条件
- 覆盖用户的**自然语言表达**，不能只依赖命令格式（如 `/save-session`）
- 同时支持命令式触发和意图式触发
- 示例：「保存会话」「记录本次会话」「帮我归档」都应能触发同一个 Skill

### 3. 细化执行步骤
- **每一步必须具体到 AI 可直接执行**，不依赖"脑补"
- 使用编号步骤（1. 2. 3.），每步包含：动作 + 预期结果
- 关键子步骤增加 checklist（如字段提取逐项核对）
- 涉及 LLM 调用的步骤需明确：temperature、超时、接口地址

### 4. 制定核心规则
- 用 **"不可违反"** 的措辞书写（如"所有 LLM 提取结果必须进入 `待提炼/`，经人工确认后才可迁移"）
- 规则数量控制在 3-6 条，过多则遵循率下降
- 包含边界规则：空产出处理、异常回退、格式异常处理

### 5. 测试验证
- Skill 编写完成后，**必须用真实场景测试**是否按规范执行
- 测试重点：触发条件是否灵敏、执行步骤是否完整、输出格式是否符合模板
- 测试通过后标记状态为 `active`，测试中发现的问题进入步骤 6

### 6. 迭代优化（分级修改）
基于使用反馈或 Brainstorming 分析，按优先级批量修改：

| 优先级 | 判定标准 | 处理策略 |
|--------|----------|----------|
| **P0** | 导致功能失败或数据错误 | 立即修复 |
| **P1** | 影响体验或效率，但有 workaround | 计划修复 |
| **P2** | 增强功能，不影响当前使用 | 暂缓，评估 token 成本/可迁移性后再决策 |

- 修改顺序：**先 SKILL.md → 再 bootstrap 同步文件**，避免遗漏
- 每次修改后用 `StrReplaceFile` 做局部替换，避免整文件重写风险

## 特化选项（视情况而定，仅部分记录涉及）

| 选项 | 适用场景 | 来源记录 |
|------|----------|----------|
| **Prompt 工程分层** | Skill 涉及 LLM 调用时，system prompt 分 5 层（角色→输入→目标→格式→边界） | Skill定义与Prompt工程 |
| **confidence 机制** | 需要 LLM 自我评估输出质量时，定义 high/medium/low 三档 | Skill定义与Prompt工程 |
| **变量标注规范** | Prompt 模板中需替换参数时，用 `{{变量名}}` 标注 | Skill定义与Prompt工程 |
| **有效提示词筛选** | 会话记录类 Skill 需区分"实质性任务"与"噪声消息"（触发词/追问/确认/反馈） | 修改会话记录 Skill |
| **自动提取 + 确认** | 信息可从上下文中推断时，自动提取后展示摘要请用户确认，而非中途提问 | 修改会话记录 Skill |
| **JSON 改 Markdown** | LLM 输出格式不稳定时，取消 JSON 解析，直接要求输出 Markdown + YAML frontmatter | 批量修改 Skill P0+P1 |
| **增量扫描** | 周期性 Skill（如 pattern-miner）需避免全量扫描时，引入 `last_scan` 时间戳 | 批量修改 Skill P0+P1 |
| **智能去重** | 聚合多条记录时，同一任务不同阶段合并为一条并标注进度演变 | 批量修改 Skill P0+P1 |

## 常见误区（多条记录都踩过）

1. **执行步骤不够具体，AI 会"脑补"**
   - 错误："扫描原料文件并提取资产"
   - 正确："扫描 `00-Inbox` 和 `01-Work` 下所有 `.md` 文件 → 排除已标记 `status: 已提炼` 的原料 → 对正文 < 200 字的直接跳过"

2. **向用户提问采集信息与 AI 工具交互模式冲突**
   - 错误：Skill 执行中途弹出选项式提问打断工作流
   - 正确：从会话上下文中自动提取 → 展示摘要 → 用户确认（一键通过或修正）

3. **触发词只覆盖命令格式，忽略自然表达**
   - 错误：只识别 `/save-session`
   - 正确：同时识别「保存会话」「记录本次」「归档一下」等同义表达

4. **提示词/消息未区分有效与噪声**
   - 错误：将会话中所有用户消息一视同仁归档
   - 正确：建立判断标准表格，过滤触发词、追问、操作指令、纯确认、纯反馈

5. **修改 Skill 时未同步关联文件**
   - 错误：只改 `SKILL.md`，忘记同步 `bootstrap/` 下的引导文档
   - 正确：修改顺序固定为 SKILL.md → bootstrap → templates → README

6. **批量修改不分优先级**
   - 错误：看到问题就改，导致改动过大难以回滚
   - 正确：先分级（P0/P1/P2），P0 立即改、P1 计划改、P2 评估成本后决定
