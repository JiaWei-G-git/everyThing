# 完成前验证

## 触发条件
在即将声称工作完成、修复成功或通过验证时，**在提交或创建 PR 之前**使用。

## 执行步骤

1. **识别验证命令**
   - 明确什么命令能证明当前声明（如 `npm test`、`pytest`、`cargo check`）
   - 确保命令覆盖本次变更的所有相关测试和检查
   - 预期结果：确定具体的验证命令列表

2. **执行完整验证**
   - 在干净环境中运行完整命令（不使用缓存、不跳过步骤）
   - 记录完整的命令输出，包括 stdout、stderr 和退出码
   - 预期结果：获得未经截断的原始输出

3. **检查结果**
   - 完整阅读输出，统计失败数量、警告数量
   - 确认退出码为 0（或预期成功码）
   - 检查是否有隐藏的失败（如测试通过但覆盖率下降）
   - 预期结果：明确的成功/失败判定

4. **确认声明**
   - 只有当输出完全确认声明时，才做出完成声明
   - 如果验证失败，返回调试阶段，不声明完成
   - 预期结果：基于证据的可靠完成声明

## 铁律
```
没有新鲜验证证据，就没有完成声明
```
