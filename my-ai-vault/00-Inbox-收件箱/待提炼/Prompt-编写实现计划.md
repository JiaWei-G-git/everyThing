# 编写实现计划

## 场景
当拥有明确的设计规范或多步骤任务需求时，在触碰代码之前，将工作拆解为粒度可控、可验证的子任务。

## 模板

请基于以下设计规范，编写一份详细的实现计划。

**设计规范**：
{{设计文档路径或内容摘要}}

**计划要求**：
- 每个步骤必须是可在 2-5 分钟内完成的单一动作
- 假设执行者对该代码库一无所知且代码品味有问题
- 记录执行者需要知道的一切信息

**计划文档头部**：
```markdown
# {{功能名称}} 实现计划

> **对于代理工作者：** 必需子技能：使用子代理驱动开发

**目标：** {{一句话描述构建什么}}

**架构：** {{关于方法的 2-3 句话}}

**技术栈：** {{关键技术/库}}
```

**每个任务必须包含**：
1. 创建/修改的确切文件路径
2. 该步骤的完整代码（或代码结构）
3. 带有预期输出的确切验证命令
4. 提交步骤（如适用）

**输出格式**：
按执行顺序列出所有任务，每个任务编号并包含上述四项内容。任务之间应有明确的依赖关系说明。
