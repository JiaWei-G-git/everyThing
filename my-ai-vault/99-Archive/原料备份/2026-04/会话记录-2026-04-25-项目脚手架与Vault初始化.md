# 会话记录 - 项目脚手架搭建与 Vault 初始化

## 背景

架构设计完成后，需要生成项目的初始文件结构。包括 Python 源码、Markdown 模板、Skill 定义和文档。计划用 subagent 驱动开发，1 天完成 Hackathon MVP。

## 用户提示词序列

```
[1] 帮我搭建项目脚手架，包括 src/ 目录、templates/ 目录、Vault 目录结构

[2] 需要创建 session_recorder.py、asset_extractor.py、vault_initializer.py 三个核心脚本

[3] 同时生成 Markdown 模板：session_record.md、weekly_report.md、prompt_asset.md

[4] 创建三个 Skill 的 SKILL.md：session-recorder、asset-extractor、weekly-generator
```

## 解决思路

**项目结构**：
```
aiKnowledgeBase/
├── src/                    # Python 核心代码
│   ├── config.py
│   ├── utils.py
│   ├── vault_initializer.py
│   ├── session_recorder.py
│   └── asset_extractor.py
├── templates/              # Markdown 模板
│   ├── session_record.md
│   ├── weekly_report.md
│   └── prompt_asset.md
├── my-ai-vault/            # Vault 根目录
│   └── 30-Skills/          # Skill 定义
├── docs/                   # 文档
├── requirements.txt
├── .env.example
└── README.md
```

**关键实现**：
- `vault_initializer.py`：创建 14 个 Vault 目录
- `session_recorder.py`：交互式归档，路径记忆到 AGENTS.md
- `asset_extractor.py`：LLM 提取 + mock 回退
- `config.py`：Vault 路径常量和 LLM API 配置

## 优化经验

1. 项目脚手架用 subagent 并行开发效率很高，但要注意文件写入冲突
2. Python 脚本的 mock 模式很重要，API 额度不足时是救命稻草
3. templates/ 目录要和 src/ 分离，模板独立管理便于后续扩展
4. .env.example 要提供完整配置示例，降低新用户上手门槛
