# 周报生成技能 (Weekly Generator)

**版本**: v1.0.0
**类型**: Skill · 项目文档自动生成器

当用户说 **"生成本周周报"**、**"写周报"** 或 **"/weekly"** 时触发此技能。

---

## 功能说明

按项目聚合 `01-Work-工作记录/` 下的会话记录与日志，调用 LLM 生成符合模板的周报或项目情况文档。支持自定义周报模板。

---

## 执行步骤

### 1. 读取项目级配置

检查当前工作目录下的 `AGENTS.md` / `CLAUDE.md`，读取：
- `session_archive_path` — 确定项目归档根目录
- `weekly_template` — 自定义周报模板路径（可选）

```markdown
<!-- AI-Vault-Config -->
- session_archive_path: "01-Work-工作记录/XX项目/会话记录/"
- weekly_template: "90-Templates/周报/敏捷开发-周报模板.md"
```

### 2. 确定项目与周期

**交互式确认**：
- 项目名（从路径中推断或询问用户）
- 周期：日 / 周 / 月（默认周）
- 时间范围（默认最近 7 天）

### 3. 聚合原料内容

扫描 `01-Work-工作记录/<项目名>/` 下的内容：
- 会话记录（`会话记录/`）
- 工作日志（`日志/`）
- 已有周报（`文档/` 下的历史周报，用于增量参考）

按以下维度聚合：
- 完成事项（Completed）
- 进行中事项（In Progress）
- 阻塞项与风险（Blockers）
- 关键决策与方案（Key Decisions）
- 下阶段计划（Next Plan）
- 资产沉淀清单（本周提炼了哪些新资产）

### 4. 选择周报模板

**模板优先级**：
1. 项目级配置中的 `weekly_template`
2. `90-Templates/周报/<项目类型>-周报模板.md`
3. `90-Templates/周报/默认周报模板.md`

**模板存放规范**：
```
90-Templates/周报/
├── 默认周报模板.md
├── 敏捷开发-周报模板.md
├── 客户项目-周报模板.md
└── 技术调研-周报模板.md
```

**模板语法**：Jinja2 占位符
```markdown
## 本周进展
{{本周进展}}

## 阻塞项
{{阻塞项}}

## 资产沉淀
{{资产沉淀}}
```

### 5. 调用 LLM 生成文档

将聚合内容 + 模板发送给 LLM，生成结构化 Markdown 文档。

### 6. 质量自检（基础）

生成后执行基础检查：
- 关键字段非空校验
- 阻塞项是否有对应解决方案
- 关联资产链接是否有效（`[[WikiLink]]` 校验）

### 7. 保存并通知

**输出路径**：
```
01-Work-工作记录/<项目名>/文档/<项目名>-周报-YYYY-MM-DD.md
01-Work-工作记录/<项目名>/文档/项目情况-<项目名>.md（持续追加）
```

向用户汇报生成结果和文件路径。

---

## 核心规则（不可违反）

1. **模板可自定义** — 项目可在 `AGENTS.md` 中指定专属模板，未指定则使用默认模板；
2. **项目情况文档持续追加** — 非覆盖，每次生成追加到现有文件末尾；
3. **周报独立文件** — 每次生成独立文件，按日期命名；
4. **自定义维度支持** — 项目可在配置中定义额外聚合维度（如"客户需求变更"、"上线风险"）；
5. **无内容不硬编** — 如某周期内无会话记录，如实汇报"本周无记录"，不编造内容。

---

## 使用示例

**用户**: "生成本周周报"

**执行**:
1. 读取项目级配置，确定项目名和模板；
2. 扫描最近 7 天的会话记录和日志；
3. 按维度聚合内容；
4. 调用 LLM 生成符合模板的周报；
5. 执行基础质量检查；
6. 保存到 `01-Work/<项目名>/文档/`；
7. 通知用户。
